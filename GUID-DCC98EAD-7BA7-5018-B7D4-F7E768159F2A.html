Category: USER-EXEC

3

In the Symbian platform 8.1, 9.0, 9.1 and subsequent versions:

  • This panic is raised when an exception is raised on the current thread by a call to User::RaiseException(), and the thread has no exception handler to handle the specified exception.

  • This panic can also be raised implicitly through a processor or hardware exception. A common case is EExcIntegerDivideByZero, created by attempting to divide by zero. See TExcType.

In the Symbian platform up to 8.1a:

  • This panic is raised when the current thread calls RThread::RaiseException() on itself, and the thread has no exception handler to handle the specified exception.

Note: the RThread::RaiseException() API is deprecated in the Symbian platform 8.1b, 9.0, 9.1 and subsequent versions, and should not be used.
