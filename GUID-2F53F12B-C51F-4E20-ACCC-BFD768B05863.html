Building the Test Product

You must first build the test product component before you run the tests.

  1. Open the command prompt and navigate to …\sip\group\.
  2. Run the following commands from the command prompt:

    bldmake bldfiles

    abld test build [winscw|armv5] [urel|udeb]

  3. Build Additional Components

    This test component is dependant upon the Testproduct ECom test plug-in:

    …\syslibsapitest\syslibssvs\TestPlugin\group\

    The plug-in is built together with the ECom test suite. There is no need to build the plug-in separately.

    When the plug-in is built, two files, Testproduct_Plugin.dll and Testproduct_Plugin.rsc, appear on z:\ecom\ of the device under test.
