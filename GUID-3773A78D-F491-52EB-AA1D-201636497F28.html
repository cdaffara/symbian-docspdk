Power Management Tutorials

This topic describes how to implement a power controller in a base port.

A power controller is a kernel extension that manages the power management hardware of a phone, for example on-chip power management controllers and oscillators.

  • Porting the Power Resource Manager
    This tutorial describes the steps needed to successfully port the PRM for a particular device.
  • Power Controller DPowerController Implementation Tutorial
    This topic describes how to implement the DPowerController class to provide the main power controller functionality in a base port.
  • Peripheral Driver Power Implementation Tutorial
    Describes an implementation of the DPowerHandler class.
  • Controllable Power Resources
    This topic explains how power resources were controlled in older releases of Symbian platform. For later releases see the Power Resource Manager documentation.
  • Peripherals As Power Resources
    Peripherals that provide services to other peripherals can be considered as power resources from the point of view of the client driver.
  • Inactivity Monitoring and Peripheral Low Power States
    Peripheral inactivity can be defined as the state when that peripheralâ€™s driver has no pending request and is not performing background tasks.
  • Peripheral Power Domains
    A peripheral power domain can be defined as being a peripheral, or group of peripherals, whose power supply can be controlled independently from the rest of the device, but not independently from other peripherals within the same power domain.
  • Additional System Wide Power States
    A base port can add new power states to improve power management for phone hardware.
  • Battery Monitor Implementation Tutorial
    A base port can implement code to monitor the battery of the phone and to provide notifications when the battery power is low.
  • Power HAL Handler Tutorial
    User-side programs can get and set a number of system attributes about power using the HAL interface.
